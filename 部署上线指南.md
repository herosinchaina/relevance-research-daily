# 把网页发布到网上（其他机器也能打开）

推荐用 **Netlify Drop**：不用注册、不用装工具、不用命令行，把文件夹拖进网页就出网址，自带 HTTPS。全程约 1 分钟。

## 方式一：Netlify Drop（最省事，推荐）

1. 浏览器打开 **https://app.netlify.com/drop**
2. 打开你的文件夹 `Desktop/codex project/relevance-research-daily`
3. 把**整个 `relevance-research-daily` 文件夹**直接拖到网页中间的虚线框里
4. 等几秒，页面顶部会出现一个网址，形如 `https://随机名字.netlify.app`
5. 点开就是你的网页。把这个网址发到任意机器/手机都能打开 ✅

> 想要好记的网址：拖完后页面上有 “Site configuration → Change site name”，可以改成 `query-relevance.netlify.app` 之类。
> 想要别人不能随便看？免费版可在站点设置里加访问密码（Visitor access / password protection）。

以后**更新论文**：改完 `data/papers.json`，回到这个站点的 “Deploys” 页，再把文件夹拖一次（覆盖部署），网址不变、内容更新。

## 方式二：GitHub Pages（适合你本来就用 GitHub）

前提：本机已装 git，有 GitHub 账号。

```bash
cd "Desktop/codex project/relevance-research-daily"
git init
git add .
git commit -m "relevance research daily"
# 在 GitHub 网页上新建一个空仓库 relevance-research-daily，然后：
git remote add origin https://github.com/你的用户名/relevance-research-daily.git
git branch -M main
git push -u origin main
```

然后在仓库页面 **Settings → Pages → Source 选 main 分支 / 根目录 → Save**，
稍等一两分钟，会给出 `https://你的用户名.github.io/relevance-research-daily/` 这个网址。

---

## 方式三：Vercel（命令行一行部署）

前提：本机已装 Node.js。

```bash
cd "Desktop/codex project/relevance-research-daily"
npx vercel
# 首次会让你登录(浏览器点一下)，之后一路回车，结束后给你一个网址
```

更新内容后再跑一次 `npx vercel --prod` 即可。

---

## 说明

- 三种方式都是「静态托管」，网页里的 `fetch('data/papers.json')` 在 http/https 下能正常读取，所以联网访问显示正常。
- 项目里的 `data/papers.inline.js` 是给「本地双击打开」兜底用的，部署上线后不影响、可保留。
- 不想公开数据的话，优先选 Netlify 并开启访问密码，或 GitHub 用 private 仓库（注意 private 仓库的 Pages 可能需要付费套餐）。
