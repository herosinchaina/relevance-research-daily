# Query-物料相关性论文库 · Query–Material Relevance

每周整理 query 与文档/网页/商品/广告等二元物料「相关性判别」方向的最新论文（每周日早上 6 点自动更新）。论文以**综述/总结 + 小白详解**形式呈现，按板块分页浏览（点哪个板块只看该板块），并**按日期归档**——右上角选择日期即可回看历史。

## 目录结构

```
relevance-research-daily/
├─ index.html                 # 网页主文件（前端源码：HTML+CSS+JS 全在这里）
├─ build.py                   # 把按日数据内嵌进 inline.js（双击打开用）
├─ new_day.py                 # 新增/重建一天后，重建索引并 build
├─ data/
│  ├─ categories.json         # ★ 板块定义（所有日期共用）
│  ├─ index.json              # 自动生成：可用日期列表（倒序）
│  ├─ papers.inline.js        # 自动生成，请勿手改
│  └─ days/
│     ├─ _template.json       # 新增一天时复制它
│     ├─ 2026-06-28.json      # ★ 每期一个文件：当期的 digests + papers
│     └─ ……
└─ README.md
```

> 旧的 `data/papers.json` 是改版前的单文件数据，保留作备份，现在网页已不读它。

## 历史论文在哪里？

**每一天的论文都永久保存在 `data/days/{日期}.json` 里，不会丢。** 新的一天开始后：

- 之前的论文仍在各自的日期文件里；
- 打开网页，**右上角有个「📅 选择日期」下拉**，默认显示最新一天，切到哪天就看哪天的论文和综述；
- 底部会显示「当前查看：某日 · 共收录 N 天」。

## 当前板块

增删板块：编辑 `data/categories.json` 的 `categories`（所有日期共用同一套板块）。

## 怎么打开

1. **直接双击 `index.html`** — 读取 `data/papers.inline.js` 内嵌的全部历史数据（运行过 build.py 即可）。
2. **本地服务器**（改完数据立即生效）：
   ```bash
   cd relevance-research-daily
   python3 -m http.server 8000   # 浏览器打开 http://localhost:8000
   ```

## 每周怎么新增一期

1. 复制 `data/days/_template.json` 为 `data/days/当周日期.json`（如 `2026-07-05.json`）。
2. 填入当期的 `digests`（每个板块一段本周综述）和 `papers`（每篇含 category / title / source / venue / tag / summary / contrib / explain / url）。`category` 必须是 `categories.json` 里的某个 id。
3. 运行：
   ```bash
   python3 new_day.py
   ```
   它会自动扫描 `data/days/` 重建 `index.json`，并重新生成内嵌数据。完成后日期下拉里就多了这一期。

> 最省事的方式：已配置每周日早上 6 点的定时任务自动整理；也可随时对我（Claude）说「整理本周的论文」，我会联网找最新论文、写好每篇的综述与小白详解、每类的本周综述，新建日期文件并 build 好。

## 每篇 / 每期的字段

- 每篇论文：`summary`（卡片正文）、`contrib`（核心贡献）、`explain`（小白详解，按【背景问题】【核心思路】【创新点】【为什么重要】【打个比方】【局限】分段）、`url`（原文链接）。
- 每期的 `digests`：每个板块一段本周综述，**必须讲到本期该板块全部论文、每篇都点名**，讲清出发点与创新点，整段不少于约 1500 字；用空行分段，`【小标题】`会在页面高亮。综述过长时网页会自动折叠，可点「展开全文」。

## 设计

杂志卡片风格，板块用颜色区分。单类别视图（一次只看一个方向）+ 本周综述置顶（过长自动折叠）+ 每篇「📖 小白详解」弹窗 + 右上角日期选择器回看历史。

## 全库论文搜索

页面顶部有一个搜索框，支持**对全部历史数据**（所有日期、所有板块）做**模糊搜索**：

- 匹配范围是全文——标题、机构、出处、标签、总结、核心贡献、小白详解都会被搜；
- 支持**多关键词**：空格分隔，需全部命中（如「电商 强化学习」只返回同时含两词的论文）；
- 大小写不敏感，输入即搜；命中的关键词会**高亮**；
- 结果以卡片平铺，每张标注**日期和板块**，可直接看小白详解、原文，或点「在某日中查看」跳回该天该板块的卡片；
- 清空搜索框即回到常规的按日期/板块浏览视图。

## 综述里的论文跳转

每期综述里提到的论文名是**可点击的**（虚线下划线）。点击后：

- 若是**当天**的论文 → 自动切到它所在板块、滚动到对应卡片并高亮闪烁；
- 若引用的是**以前某天**的论文 → 先自动切到那一天，再切板块、滚动高亮。

数据里用 `[[id|显示名]]` 标记当天论文、`[[日期:id|显示名]]` 标记历史论文（如 `[[2025-06-28:colbert|《ColBERT》]]`）。每篇论文都有稳定的 `id` 字段，跳转就靠它。写当天综述时，如果某篇是**前人工作的优化/系列续作**，就用这种标记引用历史论文，读者一点即达。

## 两项自动维护机制（在 `new_day.py` 里，每次运行自动执行）

### 1. 跨全库查重（防止重复上传）
新增一天时，会拿当天每篇论文与**历史所有日期**的论文比对，判定重复的依据：
- 归一化标题相同/高度相似（去空格标点、去掉 arXiv/会议/版本号等噪声）；
- 提取到的 **arXiv 编号相同**；
- 同一篇论文的「会议/期刊正式版」与「arXiv 预印版」会被识别为重复。

发现重复会**警告并列出**（不自动删，由你核对后手动决定），避免同一篇论文换个出处被重复收录。也可单独跑：`python3 maintain.py check 2026-06-29`。

### 2. 存储占用预估
- 每篇论文约 **2.8 KB**（其中小白详解 explain 占约 71%）。
- 每周每类最多 50 篇、5 类共最多约 250 篇 ≈ **0.7 MB/周**（实际因只收新论文、不凑数，通常远小于此）。
- 一年约 52 期，满打满算约 **35 MB/年**；若每期实际几十篇，则**一年十几 MB 量级**。
- 即便如此，**数据永久保留**、不做删除，存储压力很小。
